# 和 AI Agent 一起做事

## 一门教人把 AI 用到具体事情里的课

我身边不少朋友已经在用 ChatGPT、Gemini、豆包或 DeepSeek。平时让它改改文字、查点资料，偶尔也拿来问代码。可一到稍微完整一点的任务，他们还是会把工作切成很多小段，一句一句地喂给 AI。

我教他们用 Codex 时，最明显的变化不在模型，而在用法。AI 可以先读项目、列计划，再改文件、跑命令、看报错。处理表格、整理资料、做旅行计划也是类似的。你可以把整件事交代清楚，让它连续做一段，而不是每次只问一句。

这门课想把这种用法讲明白。

## 谁可以来学

年龄不是主要门槛。

青少年可以做自己的网页、小游戏、故事书和学习助手。成年人更可能拿它处理 Excel、写报告、整理资料，或者搭一个只有自己会用的小工具。家长和老师则需要弄清楚另一件事：孩子用 AI，重点不该是抄答案，而是学会描述想法、看懂结果，并且愿意把第一版改掉重来。

不要求会编程。会用电脑、手上有一件想做的事，就够开始了。

## 课里会做什么

我们会练习怎样把一个含糊的念头说清楚，怎样让 AI 先问问题、列计划，再开始执行。还会接触文件、表格、代码和浏览器这些工具，但不会一上来堆术语。

课里的项目会尽量贴近生活：个人主页、照片浏览器、旅行计划、学习助手、Excel 分析流程，都可以。选什么项目不重要，重要的是你真的需要它，也愿意检查 AI 做出来的东西。

我们还会讨论几个不太舒服但绕不开的问题：AI 会胡说怎么办？哪些资料不能上传？什么时候应该让它停下来？作业和工作成果的责任算谁的？

## 一个简单例子

很多人会这样问：

```text
帮我做一个网页。
```

如果这是你自己的项目，可以多交代几句：

```text
我想做一个个人主页，放摄影作品和最近写的文章。
主要给朋友和可能合作的人看。
页面需要自我介绍、作品区、博客链接和联系方式。
我喜欢干净一点的排版，不想做得像公司官网。

先别急着写代码。请检查我还漏了什么，问完问题后再列一个实现计划。
```

第二种写法没有什么玄学。你只是把真实情况告诉了 AI，也给自己留了一个确认计划的机会。

## 十章内容

1. AI 不只是聊天机器人
2. 把想法说清楚
3. 先问问题，再计划和执行
4. 工具、上下文、记忆和 skill
5. 用 AI 学习学校科目和兴趣技能
6. 用 AI 做网页、小工具和其他作品
7. 用 AI 处理表格、文档和重复工作
8. 几种常见的 Agent 工作流
9. 安全、隐私和人的责任
10. 完成并展示一个自己的项目

这门课最后希望留下的，不是某个软件的操作步骤。软件更新太快，半年后菜单可能就变了。更耐用的是一套做事习惯：任务先说清楚，计划先看一遍，结果自己验收，做顺的流程再留下来下次用。
