# Agent Thinking 101: 和 AI Agent 一起做事

## 0. 课程定位

这份大纲最早按少儿编程来写。后来范围扩到了成年人，因为很多人虽然会用 ChatGPT 或 Gemini，却还是把 AI 当成一句一答的聊天工具。

课程会教人把目标、材料和限制交代清楚，让 Agent 先计划，再使用工具执行。结果做得不对，人要能看出来，也要知道怎样让它改。

同一套方法会落到不同场景：

- 青少年版：用 Agent 创造网页、小游戏、故事、学习助手和个人作品。
- 成年人版：用 Agent 处理文档、Excel、数据分析、写代码、自动化日常工作。
- 家长/教师版：理解 AI Agent 的价值、边界、安全和教学方式。
- 开发者/效率工具用户版：理解 Codex、AI Coding Agent、skills、MCP 和个人工作流。

## 1. 为什么不只做“少儿编程”，而是“Agent 思维”

传统少儿编程通常围绕 Scratch、Python、C++、竞赛或计算思维展开。它的价值仍然存在，但 AI 时代的新问题变成了：

- 孩子不一定先学完语法，才能做出作品。
- 成年人不一定是程序员，也可以用 Agent 完成复杂工作。
- 学习的重点不只是“我会不会写代码”，而是“我能不能把想法变成可执行任务”。
- AI 不只是回答问题，而是可以参与计划、执行、检查、迭代。

成年人也常停在局部提问、局部改写和局部总结。课程因此不绑定某个年龄，也不绑定某个工具。它主要练一件事：

> 把模糊想法变成清晰目标，把清晰目标拆成步骤，把步骤交给 AI 和工具协作执行，再用人的判断检查结果。

## 2. Agent 相比普通聊天 AI 的进步

普通聊天 AI 的典型使用方式是：用户问一句，AI 回一句。它可以解释概念、写文本、生成代码片段，但大多数时候停留在“对话”和“生成内容”。

Agent 的进步在于它更接近“做事系统”：

- **目标感**：Agent 可以围绕一个目标连续工作，而不是只回答单个问题。
- **任务拆解**：Agent 可以把复杂任务拆成多个步骤，并动态调整计划。
- **工具调用**：Agent 可以使用搜索、代码执行、文件读写、浏览器、日历、表格、数据库等工具。
- **环境反馈**：Agent 会根据工具结果、报错、页面变化、文件内容继续判断下一步。
- **记忆与上下文**：Agent 可以保留偏好、项目背景、历史决策，让后续协作更顺。
- **检查与迭代**：Agent 可以生成结果，也可以检查结果、修复问题、总结经验。
- **半自动执行**：人提出目标和边界，Agent 在中间承担大量执行劳动。

- 聊天 AI 更像“会说话的老师或助手”。
- Agent 更像“能拿工具干活的项目搭子”。

具体到使用上，就是会准备目标和上下文，知道它能用哪些工具，也提前想好怎么检查。

## 3. 公开 Agent 教程调研摘要

目前公开的 Agent 教程大多偏技术实现，适合开发者，但不一定适合青少年、家长或普通职场人。

### 3.1 Hugging Face Agents Course

Hugging Face 的 Agents Course 是完整的开发者课程，覆盖 Agent 基础、框架、Agentic RAG、最终项目和认证。它会讲 tools、thoughts、actions、observations、smolagents、LlamaIndex、LangGraph 等内容。

对我们的启发：

- 它证明 Agent 教育已经是一个明确方向。
- 它强调理论加实践，最后要做项目。
- 但它要求基础 Python，更偏开发者，不适合作为青少年或普通人的第一门课。

### 3.2 Anthropic: Building Effective Agents

Anthropic 把 agentic systems 分成 workflow 和 agents。workflow 是预先写好的路径，agent 是模型根据任务动态决定流程和工具使用。它也强调一个重要原则：尽量从简单、可组合的模式开始，而不是一开始堆复杂框架。

对我们的启发：

- “简单可组合”非常适合教学。
- 青少年和成年人不需要一开始学习复杂框架，可以先理解 prompt chaining、routing、parallelization、orchestrator-workers、evaluator-optimizer 这些工作模式。
- Agent 教育要强调检查点、环境反馈和人类判断。

### 3.3 OpenAI Agents / Tools 相关文档

OpenAI 的开发者文档列出了 Agents SDK、工具调用、MCP、Computer Use、File Search、Code Interpreter 和 Shell。模型接上这些工具以后，才能读文件、操作环境并继续执行。

对我们的启发：

- 教程里要让学习者看到“工具”这个概念。
- 对青少年，可以用浏览器、网页生成、图片、文档作为工具入口。
- 对成年人，可以用 Excel、文件、代码、搜索、日程、自动化作为工具入口。

## 4. 本教程和现有教程的区别

现有教程常见的门槛：

- 太偏技术框架，普通人看不懂。
- 太偏 prompt 模板，无法迁移到真实任务。
- 太偏工具介绍，今天讲一个软件，明天软件就变了。
- 太少讨论人的角色：目标、判断、边界、审美、责任。

这份教程的取舍：

- 先讲思维，再讲工具。
- 先讲真实任务，再讲技术概念。
- 先让学习者做出作品，再逐步解释背后的原理。
- 既适合青少年创造作品，也适合成年人处理工作。
- 强调人和 Agent 的协作关系，而不是把 AI 当成答案机器。

## 5. 为什么要教这个：作者真实场景

这套教程可以从一个很真实的观察开始：

很多人已经在用 Gemini、ChatGPT、豆包、DeepSeek 或 GLM，但仍然停留在比较基础的使用方式上。他们会让 AI 改一段文字、写一个小段落、总结一篇文章，或者回答一个局部问题。这些用法当然有价值，但它们只用到了 AI 的一小部分能力。

现在更重要的变化是：AI 已经可以承担一段相对独立的工作。只要人能把目标、背景、约束和检查标准说清楚，Agent 就可以帮人拆任务、查资料、读文件、写代码、改错误、整理输出，甚至把重复的电脑操作沉淀成 skill 或工作流。

这也是本教程想降低的门槛：

- 不让高昂订阅费、复杂环境和信息差，把普通人挡在 AI 能力之外。
- 不让用户只停留在“让 AI 改一段文字”的阶段。
- 让学习者意识到，自己电脑上的很多重复劳动，都可以被整理成可复用的 Agent 工作流。
- 让青少年知道，AI 不只是帮忙写作业，而是可以帮他们创造作品。
- 让成年人知道，AI 不只是聊天工具，而是可以处理工作、学习和生活的协作者。

### 5.1 教朋友使用 Codex

一个重要起点是：身边朋友可能已经会用 ChatGPT 或 Gemini，但还没有形成 Codex / Agent 式的工作方式。

他们习惯把任务切得很碎，一段一段交给 AI，例如：

- 改一段文案。
- 翻译一段内容。
- 总结一个材料。
- 写一点代码片段。

但 Agent 式的方式是把完整目标交给 AI：

- 帮我理解这个项目，并找出我应该从哪里改。
- 帮我写一个小工具，运行起来并自己检查报错。
- 帮我把这个重复工作做成脚本或 skill。
- 帮我读这些资料，总结成可执行计划。

这里借 Codex 讲的是怎样让 AI 进入项目现场。以后工具换了，这套用法仍然能迁移。

### 5.2 把重复工作做成 skill 或工作流

很多人的电脑里都有重复劳动：

- 每周整理表格。
- 固定格式写报告。
- 批量处理图片或文件。
- 从网页收集信息。
- 把资料转成笔记。
- 把某类任务拆解成固定流程。

传统做法是每次手工做一遍。Agent 思维会问：

> 这件事能不能被描述成一套稳定流程？能不能让 AI 先学会规则，再帮我半自动执行？

这可以进一步变成 skill、脚本、模板或个人工作流。普通人不一定要先成为程序员，也可以先学会识别“哪些任务值得自动化”。

### 5.3 用 AI 做个人复盘和自我理解

Agent 不只可以处理外部任务，也可以帮助人整理自己。

例如把生平经历、随想、博客、日记、长期写作、重要选择、情绪记录和价值观材料整理给 AI，让它形成一份对自己的调研：

- 我长期关心什么？
- 我的表达里反复出现哪些主题？
- 我的优势和困境是什么？
- 我最近的拖延、焦虑或兴奋来自哪里？
- 我适合怎样的学习和工作节奏？
- 我过去的经历可以沉淀成什么项目？

AI 可以先把散落的材料整理出来，像一面会分类的镜子。结论仍然要自己判断，尤其是涉及性格和人生选择时。

### 5.4 用 AI 学习兴趣和技能

AI 可以成为一个非常灵活的私人教练。

例如：

- 学摄影：让 AI 分析照片风格、设计练习任务、解释构图和后期思路。
- 学化妆：让 AI 根据脸型、场景、预算和已有工具设计学习路线。
- 学投资：让 AI 从基础概念、风险意识、资产配置和案例拆解开始教，而不是直接推荐买什么。
- 学编程：让 AI 根据目标项目倒推所需知识。
- 学任何新领域：让 AI 设计课程、出练习、检查理解、安排复习。

这里的心法是：不要只问“这个概念是什么”，而是让 AI 变成一个能持续追踪目标、生成练习、调整难度的学习系统。

### 5.5 用 AI 做旅行计划和生活计划

旅行计划也是很好的 Agent 教学案例，因为它天然包含目标、约束、偏好、资料、路线和迭代。

普通问法是：

> 帮我做一个东京三日游攻略。

Agent 式问法是：

> 我想做一个东京三日游计划，预算中等，喜欢摄影、咖啡、城市散步，不喜欢太赶。请先问我 5 个必要问题，再按每天上午、下午、晚上设计路线，并说明为什么这样安排。最后给我一个可调整清单。

旅行计划很适合入门，因为偏好、预算和路线随时会变。Agent 要根据这些变化反复调整，不能只生成一份漂亮攻略。

### 5.6 用 AI 搭自己的小工具

Agent 最有冲击力的场景之一，是让普通人意识到：

> 我有一个小需求，不一定要等别人做软件，我可以让 AI 帮我搭一个够自己用的小工具。

例如做一个本地照片库工具：

- 从下载的照片里随机展示 9 张。
- 用户可以快速浏览。
- 喜欢的照片可以 star。
- 之后可以回看收藏。

这个工具不一定复杂，但意义很大。它让学习者理解，AI 可以把“我脑子里一个很小的需求”变成“电脑上一个真的能用的东西”。

青少年可以用它做个人主页、小游戏、故事书、兴趣网站。成年人可以用它做数据整理、图片管理、资料归档、工作自动化。

## 6. 反复练习的习惯

### 6.1 从“提问者”变成“任务设计者”

普通用法是问：

> 帮我做一个网页。

Agent 思维是描述：

> 我想做一个展示我篮球爱好的个人主页。目标用户是同学和家长。页面需要有自我介绍、喜欢的球星、训练记录和照片区域。请先帮我拆需求，问我缺少的信息，再给出实现计划。

多写的这些信息，会减少 AI 自己猜测的空间。

### 6.2 从“要答案”变成“要过程”

普通 AI 使用者容易让 AI 直接给最终答案。Agent 思维会要求 AI 暴露中间过程：

- 先问我需要补充的信息。
- 先列计划，不要直接执行。
- 每一步完成后告诉我检查方法。
- 如果不确定，给出假设。
- 如果失败，解释原因并提出下一步。

这会培养学习者的过程意识，也能减少盲目信任 AI。

### 6.3 从“复制粘贴”变成“人机迭代”

AI 的第一版通常只是草稿。接下来要做的是：

1. 人提出目标。
2. Agent 给出方案。
3. 人指出不满意的地方。
4. Agent 修改。
5. 人检查结果。
6. Agent 总结经验。

用几次以后，学习者会自然形成自己的检查清单。

### 6.4 从“学工具”变成“搭工作流”

工具会变，但工作流更稳定。

例如成年人处理 Excel 数据，可以拆成：

1. 说明业务目标。
2. 上传或描述数据。
3. 让 AI 检查字段和异常。
4. 让 AI 生成分析思路。
5. 让 AI 输出图表或公式。
6. 人检查结果是否符合业务常识。
7. 让 AI 写成报告。

同样的工作流可以迁移到学习计划、投资入门、论文阅读、代码开发、旅行规划。

### 6.5 从“让 AI 替我思考”变成“让 AI 放大我的思考”

这是安全边界，也是价值观。

AI 可以帮我查资料、拆任务、写草稿、做计算、找错误，但最终的目标、判断、价值选择和责任仍然属于人。

对青少年来说，这一点尤其重要：不能把 AI 当成作业代写器，而要把它当成学习教练、创作伙伴和练习生成器。

### 6.6 从“消费软件”变成“创造自己的工具”

过去很多人遇到一个需求，会先想：

> 有没有现成软件？有没有教程？有没有别人做好的工具？

Agent 时代可以多一个问题：

> 这个需求能不能让 AI 帮我做一个够用的小工具？

这不意味着每个人都要成为专业程序员，而是每个人都可以拥有更强的数字创造力。尤其对青少年来说，这种体验很关键：电脑不是只能用来消费内容，也可以用来创造自己的系统。

### 6.7 从“单次使用”变成“长期协作”

很多 AI 使用停留在一次性对话。Agent 思维更重视长期协作：

- 把偏好写成 memory。
- 把常用流程写成模板。
- 把重复任务做成 skill。
- 把项目背景沉淀成上下文包。
- 让 AI 每次做完都总结经验。

长期协作会让 AI 越来越像一个了解你的工作伙伴，而不是每次都要重新认识你的临时工具。

## 7. 课程主线

### 第一章：AI 不只是聊天机器人

目标：建立从聊天 AI 到 Agent 的基本区别。

要讲的问题：

- 什么是聊天 AI？
- 什么是 Agent？
- 为什么工具、记忆、反馈和执行很重要？
- 为什么 Agent 像一个“能干活的搭子”？
- 为什么很多人已经在用 AI，任务仍然停在一句一答？

练习：

- 同一个任务分别用“聊天方式”和“Agent 方式”提问，比较结果差异。
- 把一个“改文字”的局部任务，升级成一个完整工作流任务。

### 第二章：把想法说清楚

目标：学会把模糊想法变成 Agent 能执行的任务。

任务框架：

- 目标：我要完成什么？
- 背景：为什么要做？
- 对象：给谁用？
- 约束：时间、工具、格式、难度、风格。
- 成功标准：怎样算做好？
- 检查方式：我怎么知道它做对了？

练习：

- 把“我想做个网页”改写成一个完整任务说明。
- 把“我想学投资”改写成一个学习计划任务说明。

### 第三章：让 AI 帮你学习

目标：让 AI 成为学习教练，而不是答案机器。

应用场景：

- 学校科目：数学、英语、历史、科学。
- 兴趣学习：投资入门、摄影、设计、视频剪辑、编程。
- 课程作业：理解题意、拆解步骤、检查草稿、生成练习。

重要边界：

- 可以让 AI 讲解和训练。
- 可以让 AI 帮你检查错误。
- 不应该让 AI 直接替你完成需要自己提交的作业。

练习：

- 让 AI 为一个学习目标设计 7 天计划。
- 让 AI 根据错题生成 5 道同类练习。
- 让 AI 用三种难度解释同一个概念。

### 第四章：让 AI 帮你管理生活

目标：理解 Agent 可以成为个人操作系统的一部分。

应用场景：

- 提醒和日程。
- todo 和项目拆解。
- 读书笔记和长期记忆。
- 周复盘和目标追踪。
- 兴趣资料库。
- 个人经历整理和自我复盘。

练习：

- 设计一个“我的学习/生活助理”。
- 写一段个人偏好 memory。
- 让 AI 根据本周任务生成每日计划。
- 让 AI 根据一段自我介绍，提炼自己的长期主题和兴趣线索。

### 第五章：让 AI 帮你创造作品

目标：让青少年或初学者体验从想法到作品的完整闭环。

可选项目：

- 个人主页。
- 兴趣展示网站。
- AI 故事绘本。
- 小游戏。
- 学习打卡工具。
- 家庭提醒页面。
- 班级知识问答助手。
- 本地照片浏览和收藏工具。
- 旅行计划生成器。

项目流程：

1. 想法描述。
2. 需求拆解。
3. 页面或功能规划。
4. 生成第一版。
5. 运行和检查。
6. 修改样式和功能。
7. 展示作品。

练习：

- 做一个“关于我”的网页。
- 做一个“我的兴趣知识库”。
- 做一个“给低年级同学用的学习助手”。

### 第六章：让 AI 帮成年人干活

目标：说明 Agent 思维不只属于孩子，也能提高成年人工作效率。

应用场景：

- 处理 Excel：清洗数据、生成公式、做统计、写分析报告。
- 处理文档：总结、改写、提纲、会议纪要。
- 处理数据：读取 CSV、画图、解释趋势。
- 软件开发：写脚本、改 bug、生成网页、搭小工具。
- 调研和写作：收集资料、提炼观点、生成结构。
- 把重复工作沉淀成 skill、脚本或固定工作流。

练习：

- 给 AI 一份表格任务，让它先问清楚目标再动手。
- 让 AI 帮你把一个工作需求拆成可执行清单。
- 让 AI 写一个小脚本或网页，并教你怎么验证。

### 第七章：Agent 工作流

目标：把零散使用升级成稳定方法。

五个基础工作流：

- **链式工作流**：先大纲，再内容，再润色，再检查。
- **路由工作流**：不同问题交给不同处理方式。
- **并行工作流**：同时生成多个方案，再比较。
- **指挥者-执行者工作流**：一个 Agent 拆任务，多个子任务分别执行。
- **评审-优化工作流**：一个 Agent 生成，另一个 Agent 检查。

练习：

- 用“评审-优化”方式改一篇作文或文章。
- 用“并行方案”方式设计三个网页风格。
- 用“指挥者-执行者”方式拆一个小项目。

### 第八章：工具、记忆和上下文

目标：理解 Agent 为什么比纯聊天更强。

本章概念：

- 工具：搜索、文件、浏览器、代码、表格、日历。
- 记忆：偏好、历史、长期目标。
- 上下文：当前任务所需的背景材料。
- 权限：哪些事情可以让 AI 做，哪些必须人确认。
- skill：把反复出现的任务规则沉淀成可复用能力。

练习：

- 为自己的学习 Agent 写一份角色说明。
- 为自己的工作 Agent 写一份工具清单。
- 为一个项目写上下文包：目标、资料、规则、输出格式。
- 为一个重复任务写一份 skill 草稿。

### 第九章：安全、边界和判断

目标：建立负责任的 AI 使用习惯。

要讲的问题：

- AI 会胡说，怎么验证？
- 什么时候不能上传隐私数据？
- 什么时候不能让 AI 代写？
- 如何避免过度依赖？
- 如何让家长和老师放心？

检查清单：

- 重要事实要查来源。
- 重要计算要复核。
- 隐私信息要脱敏。
- 作业要保留自己的思考过程。
- 自动执行前要有人确认。

### 第十章：最终项目

目标：完成一个可以展示的作品或工作流。

青少年项目方向：

- 我的个人主页。
- 我的 AI 学习助手。
- 我的互动故事书。
- 我的兴趣知识库。
- 我的小游戏。

成年人项目方向：

- 我的工作自动化助手。
- 我的 Excel 分析流程。
- 我的学习课程生成器。
- 我的项目管理 Agent。
- 我的代码开发助手。

最终交付：

- 作品本身。
- 任务说明。
- Agent 对话记录摘要。
- 遇到的问题和解决方法。
- 下一步改进计划。

## 8. 四节试讲课设计

### 第 1 课：AI 不只是聊天机器人

目标：让学习者理解 Agent 和普通聊天的区别。

课堂任务：

- 用两种方式完成同一个任务。
- 比较“直接问答案”和“让 AI 先拆任务”的差别。
- 写出自己的第一个 Agent 式任务说明。
- 演示：把“改一段文字”升级成“完成一份完整材料”。

课后作品：

- 一份“我想让 AI 帮我完成的 3 件事”清单。

### 第 2 课：把一个想法变成项目

目标：学会描述目标、背景、对象、约束和成功标准。

课堂任务：

- 选择一个作品方向。
- 让 AI 帮忙提问和完善需求。
- 输出项目计划。

课后作品：

- 一份个人项目 brief。

### 第 3 课：用 Agent 做出第一个网页或小工具

目标：体验从想法到可运行作品。

课堂任务：

- 生成网页第一版。
- 检查哪里不符合预期。
- 让 AI 修改样式、文案和功能。

课后作品：

- 一个可展示的网页、小工具或故事作品。

### 第 4 课：让 AI 成为学习和生活助理

目标：把 Agent 思维迁移到长期学习和日常生活。

课堂任务：

- 设计一个学习助理或生活助理。
- 写 memory 和规则。
- 生成一周计划。
- 讨论安全边界。

课后作品：

- 一份个人 AI 使用手册。

## 9. 写作风格

教程应该尽量避免以下风格：

- 太像技术文档。
- 太像商业课招生文案。
- 太像工具说明书。
- 太多术语，太少真实例子。

更合适的风格：

- 像一个懂 AI 的朋友在教你。
- 先给真实场景，再解释概念。
- 每章都有一个能动手做的小任务。
- 每个抽象概念都配一个青少年例子和一个成年人例子。
- 保留作者自己的使用经验，比如 Codex、提醒、memory、todo、学习、写作、开发。
- 多写“我怎么真的用它”的案例，比如自我复盘、摄影、化妆、旅行计划、本地照片工具。

## 10. 后续写作顺序

建议不要从第一章开始硬写。更好的顺序是：

1. 先写“反复练习的习惯”。
2. 再写“Agent 相比聊天 AI 的进步”。
3. 再写“为什么要教这个：作者真实场景”。
4. 再写“青少年创造作品”。
5. 再写“成年人干活”。
6. 最后补齐开头、结尾和安全边界。

原因很简单：工具介绍随时可以补，作者自己的使用经验才决定这门课和其他教程有什么不同。
