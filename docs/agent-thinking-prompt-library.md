# Agent Thinking 101 Prompt 与 Skill 模板库

这些模板可以直接复制，但别把它们当咒语。先填进自己的目标、材料和限制。用过几次后，删掉多余的问题，留下每次都会用到的部分。

## 1. 通用任务说明模板

适合几乎所有任务。

```text
我想完成：

背景是：

这个结果给谁用/给谁看：

我已有的材料：

限制条件：

我希望最后输出：

怎样算做好：

请你先检查我的描述是否清楚。如果不清楚，先问我问题，不要急着开始做。
```

## 2. Agent 式协作流程模板

适合写代码、写文档、做数据、做项目。

```text
请按这个流程帮我：

1. 先复述你理解的目标。
2. 再问我 3 到 5 个必要问题。
3. 然后给出执行计划。
4. 等我确认后，只执行第一步。
5. 每一步结束后告诉我怎么检查结果。
6. 如果遇到不确定的地方，请明确说出你的假设。
7. 完成后总结这次任务可以沉淀成什么经验。
```

## 3. 让 AI 先问问题

适合需求还不清楚的时候。

```text
我现在只有一个模糊想法：

……

请不要直接给方案。
请先像一个有经验的项目顾问一样，问我 8 个关键问题。
这些问题要帮助我澄清目标、用户、材料、限制、风格、成功标准和风险。
问完后，等我回答，再帮我整理成任务说明。
```

## 4. 把想法变成项目 brief

适合青少年作品、个人项目、小工具、课程项目。

```text
请帮我把下面这个想法整理成项目 brief：

想法：

……

请输出：
1. 项目名称。
2. 这个项目解决什么问题。
3. 给谁使用。
4. 核心功能。
5. 第一版最小可行范围。
6. 需要我补充的材料。
7. 可能遇到的困难。
8. 第一版验收标准。
```

## 5. 做个人主页

适合青少年、成年人、创作者、求职者。

```text
我想做一个个人主页。

它不是简历，而是展示我这个人。

我的身份/兴趣：

我想展示的内容：

目标读者：

我喜欢的风格：

我不喜欢的风格：

请先帮我设计页面结构，再问我还缺哪些内容。
等我确认后，再生成第一版 HTML/CSS。
生成后请告诉我怎么在本地打开，以及如何检查页面有没有问题。
```

## 6. 做兴趣展示网站

适合孩子的作品课。

```text
这个网站主要给【同学/家长/朋友】看。

我希望包含：
- 我为什么喜欢它。
- 我最喜欢的作品/人物/案例。
- 我自己的经历或练习记录。
- 我想继续学习的方向。

我的技术基础是：

页面风格希望：

请先帮我拆成 3 到 5 个页面模块，然后问我还需要补充什么。
```

## 7. 做学习助手

适合学校科目、兴趣学习、成年人自学。

```text
我想做一个学习助手，用来帮我学习：

我的基础：

我每周能投入的时间：

我容易卡住的地方：

我希望它帮我：
- 制定计划。
- 出练习。
- 检查我的回答。
- 根据错误调整难度。
- 做阶段复盘。

请先帮我设计这个学习助手的工作方式。
不要直接开始讲课。
```

## 8. 让 AI 像老师一样讲题

适合学生，不适合直接代写。

```text
请你像老师一样教我这道题。

要求：
1. 不要直接给最终答案。
2. 先问我看到题目后的想法。
3. 如果我答错，先指出错在哪里。
4. 如果我卡住，只给一个小提示。
5. 最后总结这类题的解法。
6. 再出 2 道同类题让我练习。

题目是：

……
```

## 9. 设计 7 天学习计划

适合兴趣学习、技能入门。

```text
我想在 7 天内入门：

我的基础是：

我每天最多能花：

我希望第 7 天能完成一个小作品或小总结。

请帮我设计 7 天计划。
每天包括：
1. 今天学什么。
2. 一个 30 分钟以内的小任务。
3. 怎么检查我有没有学会。
4. 如果太难，可以怎么降低难度。
```

## 10. 学投资的安全模板

只用于知识学习，不用于具体投资建议。

```text
我想学习投资入门。

请注意：
1. 不要推荐具体股票、基金或买卖时点。
2. 不要替我做投资决策。
3. 请把重点放在基础知识、风险意识、资产类别、长期收益、分散配置和常见误区。
4. 每讲一个概念，都给一个生活化例子。
5. 每节最后给我 3 个问题，检查我是否理解。

请先问我当前基础和学习目标，再设计入门路线。
```

## 11. 处理 Excel 或 CSV

适合成年人工作流。

```text
我需要分析一份表格。

业务目标是：

表格字段大概包括：

我希望得到：

请不要直接下结论。
请先帮我：
1. 检查字段含义。
2. 判断可能的缺失值和异常值。
3. 提出 3 到 5 个分析维度。
4. 告诉我每个维度为什么有意义。
5. 等我确认后，再开始计算或写公式。

最后请输出一份可以给同事看的摘要，并说明哪些结论需要人工复核。
```

## 12. 写工作报告

适合周报、项目汇报、调研报告。

```text
我需要写一份报告。

报告类型：

读者：

目标：

已有材料：

希望语气：

请按流程来：
1. 先提炼材料里的关键信息。
2. 再指出缺少哪些信息。
3. 给出报告结构。
4. 等我确认后再写初稿。
5. 初稿完成后，检查空话、重复、逻辑跳跃和读者看不懂的地方。
```

## 13. 用 Codex 改项目

适合 AI Coding Agent。

```text
请先阅读当前项目结构。

我的目标是：

请先不要直接修改文件。

请先告诉我：
1. 这个项目大概是什么技术栈。
2. 与目标最相关的文件有哪些。
3. 你建议怎么改。
4. 每一步怎么验证。
5. 可能的风险是什么。

等我确认后，再开始实施。
```

## 14. 让 AI 修 bug

适合代码报错、网页问题、小工具问题。

```text
我遇到了一个问题：

现象：

我期望的结果：

实际结果：

相关报错：

我已经尝试过：

请不要直接猜原因。
请先根据现象列出 3 个可能原因，并告诉我如何验证每个原因。
然后从最可能、成本最低的验证开始。
```

```text
我的运行环境：

我希望第一版尽量简单可用，不追求复杂功能。

请先问我必要问题，再给出最小实现方案。
```

```text
目的地：

天数：

同行人：

预算：

喜欢：

不喜欢：

节奏偏好：
```

## 17. 个人复盘和自我理解

适合整理博客、日记、随想、人生经历。

```text
我会给你一些关于我的材料，包括经历、随想、博客或日记片段。

请你不要急着评价我。

请帮我做一份温和、诚实的自我观察：
1. 我反复关心的主题。
2. 我的优势。
3. 我容易卡住的地方。
4. 我最近可能在逃避或拖延什么。
5. 我可以继续发展成项目的线索。
6. 你不确定但值得我思考的问题。

请把结论和证据分开写。
不要把猜测写得太确定。
```

## 18. 评审-优化

适合文章、报告、课程、网页、prompt。

```text
请先作为作者/执行者给出第一版。

完成后，请切换成挑剔但友好的评审，检查：
1. 有没有空话。
2. 有没有逻辑跳跃。
3. 有没有太像 AI 的句子。
4. 有没有读者看不懂的术语。
5. 有没有可以删掉的段落。
6. 有没有缺少例子。
7. 有没有事实需要验证。

请先列出评审意见。
等我确认哪些意见要采纳后，再改第二版。
```

## 19. 把重复任务写成 skill 草稿

适合沉淀个人工作流。

```text
我想把一个重复任务写成 skill 草稿。

任务名称：

我为什么经常做它：

输入通常是什么：

输出应该是什么：

标准步骤：

判断标准：

容易出错的地方：

哪些步骤可以 AI 做：

哪些步骤必须人确认：

请帮我整理成一份给 AI 使用的操作手册。
要求清晰、可复用、不要写成泛泛建议。
```

## 20. 让 AI 总结经验

适合每次任务结束后沉淀。

```text
请总结这次协作。

请包括：
1. 我一开始的目标是什么。
2. 我们实际完成了什么。
3. 中间遇到什么问题。
4. 哪些 prompt 或流程有效。
5. 下次可以复用什么。
6. 如果要写成 skill，应该包含哪些规则。
7. 我作为使用者哪里可以说得更清楚。
```

## 21. Prompt 使用原则

模板只是一个起点。检查下面几项有没有说清楚：

- 目标。
- 背景。
- 材料。
- 约束。
- 成功标准。
- 检查方式。

prompt 写得长不代表任务清楚。反过来，背景已经明确时，几句话也够用。

## 22. Skill 草稿示例：旅行计划助手

```text
技能名称：旅行计划助手

用途：
帮助用户根据目的地、预算、兴趣、同行人和节奏偏好，制定可调整的旅行计划。

工作流程：
1. 先询问目的地、时间、同行人、预算、兴趣和禁忌。
2. 不要直接生成攻略。
3. 先总结用户偏好。
4. 给出 2 到 3 种旅行节奏方案。
5. 用户选择后，再按天安排上午、下午、晚上。
6. 每个安排说明原因。
7. 最后输出可调整清单。

检查标准：
- 路线不要太赶。
- 安排要符合用户兴趣。
- 不确定的营业时间、票价和交通信息要提示用户查证。
- 不替用户做涉及付款的操作。
```

## 23. Skill 草稿示例：个人写作编辑

```text
技能名称：个人写作编辑

用途：
帮助用户把口语化想法整理成博客、课程文案或长文，但保留真实感。

风格规则：
- 不要太营销。
- 不要堆概念。
- 少用口号。
- 保留一点个人观察和犹豫。
- 句子可以短一点。
- 先讲真实场景，再讲抽象概念。

工作流程：
1. 先提炼原始材料里的核心观点。
2. 问用户目标读者是谁。
3. 给 2 到 3 个切入角度。
4. 等用户选方向后再写初稿。
5. 写完后检查 AI 腔、空话、重复和逻辑跳跃。
```

## 24. Skill 草稿示例：学习教练

```text
技能名称：学习教练

用途：
帮助用户学习一个新领域，但不直接替用户完成应该自己完成的作业。

工作流程：
1. 先询问学习目标、当前基础、可投入时间和截止时间。
2. 把目标拆成阶段。
3. 每个阶段给小任务和检查方法。
4. 用户回答后再反馈。
5. 如果用户卡住，先给提示，不直接给最终答案。
6. 每周做一次复盘。

边界：
- 不代写作业。
- 不替用户做高风险决策。
- 涉及投资、医学、法律时，只做知识解释，并提醒查专业来源。
```
